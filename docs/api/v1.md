# HTTP API `/v1`

The HTTP server is an optional delivery surface over the same embedded
engine. Ordinary CLI and Rust embedding never start a listener. The canonical
machine contract is [`contracts/openapi/hyphae-v1.yaml`](../../contracts/openapi/hyphae-v1.yaml);
the checked-in JSON Schemas define every payload.

## Secure defaults

```bash
hyphae serve --data-dir ./hyphae-data
```

This binds only `127.0.0.1:8787`. A non-loopback `--bind` fails before the
socket is opened unless a bearer token is configured. Secrets are accepted
through `HYPHAE_BEARER_TOKEN` or `--bearer-token-file`; there is deliberately
no bearer-token command-line value that could leak through process listings.

```bash
umask 077
printf '%s\n' 'replace-with-at-least-32-visible-ascii-bytes' > hyphae.token
hyphae serve \
  --data-dir ./hyphae-data \
  --bind 0.0.0.0:8787 \
  --bearer-token-file ./hyphae.token
```

On Unix, token files granting any group/other permission are rejected. On
Windows, protect the file with the account ACL. Remote transport still
requires a trusted TLS-terminating boundary; Hyphae does not claim built-in
certificate management.

Health and capability routes are public and disclose no records or secrets.
When a token is configured, every KV, query, proof, and witness route requires
`Authorization: Bearer ...`. Missing and wrong tokens have the same response.

## Routes

| Method | Path | Semantics |
|---|---|---|
| `GET` | `/v1/health/live` | Process liveness |
| `GET` | `/v1/health/ready` | Owned engine is open and ready |
| `GET` | `/v1/capabilities` | Implemented features and effective limits |
| `POST` | `/v1/kv/put` | Atomic structured-record batch |
| `POST` | `/v1/kv/delete` | Atomic binary-key delete batch |
| `POST` | `/v1/kv/get` | Proven exact presence or absence |
| `POST` | `/v1/query` | Proven deterministic structured query |
| `GET` | `/v1/witnesses/{sequence}/{digest}` | Verified snapshot witness stream |

Example without authentication on the loopback default:

```bash
curl --fail-with-body \
  -H 'Content-Type: application/json' \
  -d '{"records":[{"key_hex":"616c706861","value":{"score":10}}]}' \
  http://127.0.0.1:8787/v1/kv/put

curl --fail-with-body \
  -H 'Content-Type: application/json' \
  -d '{"filter":{"op":"compare","path":["score"],"operator":"greater_or_equal","value":10},"limit":100}' \
  http://127.0.0.1:8787/v1/query
```

Every successful get/query includes canonical `.hyproof` bytes in padded
base64, its digest, a caller-pinnable anchor digest, and the authenticated
witness path. Download that exact snapshot and use `hyphae verify` for fully
offline reexecution. A server response does not replace the caller's duty to
pin or obtain the expected anchor through a trusted channel.

## Bounded behavior

`GET /v1/capabilities` reports the effective request bytes, body deadline,
JSON depth/nodes, key/document, batch, query shape/work/result, proof,
witness, response, concurrency, and query-timeout limits. The server reads a
body under byte and time bounds before starting a durable operation. Query
deadlines are cooperative inside the deterministic engine. Limit and timeout
failures return no partial logical result.

Durable engine access is admitted through a bounded semaphore and serialized
behind the single owned writer. Saturation returns `busy` with `Retry-After`.
Witnesses are verified before being streamed and are never constructed from a
caller-controlled filesystem path.

If a log commit becomes durable but its rebuildable index update fails, the
write still returns the definite durable receipt and readiness flips to
`unavailable`. Restarting the process replays the log before readiness returns;
the same transaction UUID then resolves as an exact existing commit.

Every HTTP response carries `X-Request-Id`. Every public error is JSON and
uses the same ID in its body; framework text errors are not part of the
contract. See [`error-codes-v1.md`](error-codes-v1.md).

Ctrl+C initiates graceful shutdown. No new process can open the same data
directory until the server releases its operating-system lock.
