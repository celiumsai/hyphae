# Hyphae 0.2.0 release gate

Status: local release candidate; publication not authorized

Hyphae `0.2.0` may be pushed, tagged, or published only after the local
candidate is exact, every required hosted gate passes on that exact commit,
and Mario gives explicit release authorization.

## Candidate scope

- Workspace and public package version: `0.2.0`.
- Disk format: `2`, with explicit format-1 migration.
- Public HTTP API major: `/v1`, additive retrieval contracts only.
- Retrieval semantics: exact vector v2, lexical v1, hybrid v1.
- Proof formats: existing `result-proof-v1` plus additive
  `retrieval-proof-v1`.

## Local evidence

- Rust format, clippy, workspace tests, rustdoc, and MSRV.
- Canonical contracts, generated assets, documentation examples, disk-format
  fixture, and retrieval golden synchronization.
- Rust, TypeScript, Python, remote CLI, and MCP conformance.
- Astro, Next, Vite, PliegoRS, and public-boundary checks.
- Dependency audit/deny, package-content audit, deterministic archive tests,
  native archive install smoke, load, restart/restore soak, and bounded fuzz.
- Retrieval latency, memory, write amplification, recovery, proof, and
  lifecycle measurements under [`evidence/`](evidence/).

The automated package-install smoke is the milestone flow: it runs one
installed binary, writes documents and vectors, executes exact, lexical, and
hybrid retrieval, downloads each witness, destroys the live-server dependency
for offline verification, verifies every retrieval proof, then validates
compaction and backup/restore.

## Authorization boundary

- [x] Local implementation is complete.
- [x] Re-run and record every final local gate on the versioned candidate.
- [x] Create the exact local release candidate commit.
- [ ] Receive explicit approval before any push or tag.
- [ ] Push the exact candidate and run Linux, macOS, and Windows hosted gates.
- [ ] Confirm hosted CI, Stress, Fuzz, Security, and release-candidate
  packaging on the same commit.
- [ ] Receive explicit approval before creating `v0.2.0` or publishing crates,
  SDKs, archives, signatures, attestations, or release notes.

No unchecked item may be inferred from local evidence. A successful local
build is not hosted release evidence, and a push is not publication
authorization.
