# Hyphae 0.1.0 release gate

`0.1.0` may be tagged and the repository may become public only when every
item below has current CI evidence on the release commit.

## Installation and autonomy

- [x] Clean install produces one `hyphae` binary on Linux, macOS, and Windows.
- [x] A new instance requires only a writable data directory.
- [x] KV, structured query, backup, restore, and verification work offline.
- [x] No external database, cache, cloud, LLM, or embedding provider is
      required or contacted by default.

## Durability

- [x] Atomic batches survive kill/restart at every commit boundary.
- [x] Idempotency survives restart and rejects key reuse with a new payload.
- [x] Partial tails are recovered; checksum or chain corruption fails loudly.
- [x] Snapshots, compaction, and migrations survive interruption.
- [x] Backup is consistent and restore activates only after full verification.
- [x] Compatibility fixtures open for every supported on-disk format.

## Correctness and provenance

- [x] KV, filters, sort, cursor pagination, and aggregations match the reference
      model under property tests.
- [x] Global merge precedes final limits and has deterministic tie-breaking.
- [x] Timeouts and resource limits never return silent partial results.
- [x] Semantic retrieval abstains explicitly and remains optional.
- [x] Every `/v1` get/query result includes a proof accepted by the offline
      verifier; embedded and local CLI proof-bearing paths use the same format.
- [x] Edit, delete, insert, reorder, replay, rollback, truncation, and bit-flip
      tamper cases are detected according to the documented threat model.

## Public surface

- [x] OpenAPI 3.1 and JSON Schema 2020-12 are canonical and lint clean.
- [x] The `/v1` compatibility policy and stable error codes are documented.
- [x] Loopback is the server default; remote bind requires explicit auth.
- [x] Request body, JSON depth, batch, result, timeout, concurrency, and memory
      limits have enforcement tests.
- [x] Rust, TypeScript, Python, CLI, and MCP pass the same black-box suite.
- [x] Framework integrations are optional and import no core internals.

## Supply chain and operations

- [x] Format, Clippy, tests, doctests, examples, MSRV, and docs are green.
- [x] RustSec, license/source policy, dependency review, and secret scanning are
      green.
- [x] Fuzz, fault-injection, load, and soak gates meet recorded thresholds.
- [x] Release archives contain checksums, SPDX/CycloneDX SBOM, signatures, and
      build provenance.
- [x] Installation, upgrade, migration, backup, restore, doctor, and verify
      documentation is executable from a clean environment.
- [x] Every documentation page is indexed, local links resolve, JSON examples
      parse, and the CLI command reference matches the built binary.

## Authoritative evidence

The checked boxes describe the release commit only while CI, Security,
Dependency Review, Fuzz, Stress, and Release are green on the exact same
commit. The Release matrix installs and exercises every native archive before
assembling signed artifacts. Any source change reopens every checkbox until
the full matrix passes again.
