# Hyphae 0.1.0 release gate

`0.1.0` may be tagged and the repository may become public only when every
item below has current CI evidence on the release commit.

## Installation and autonomy

- [ ] Clean install produces one `hyphae` binary on Linux, macOS, and Windows.
- [ ] A new instance requires only a writable data directory.
- [ ] KV, structured query, backup, restore, and verification work offline.
- [ ] No external database, cache, cloud, LLM, or embedding provider is
      required or contacted by default.

## Durability

- [ ] Atomic batches survive kill/restart at every commit boundary.
- [ ] Idempotency survives restart and rejects key reuse with a new payload.
- [ ] Partial tails are recovered; checksum or chain corruption fails loudly.
- [ ] Snapshots, compaction, and migrations survive interruption.
- [ ] Backup is consistent and restore activates only after full verification.
- [ ] Compatibility fixtures open for every supported on-disk format.

## Correctness and provenance

- [ ] KV, filters, sort, cursor pagination, and aggregations match the reference
      model under property tests.
- [ ] Global merge precedes final limits and has deterministic tie-breaking.
- [ ] Timeouts and resource limits never return silent partial results.
- [ ] Semantic retrieval abstains explicitly and remains optional.
- [ ] Every result includes a proof accepted by the offline verifier.
- [ ] Edit, delete, insert, reorder, replay, rollback, truncation, and bit-flip
      tamper cases are detected according to the documented threat model.

## Public surface

- [ ] OpenAPI 3.1 and JSON Schema 2020-12 are canonical and lint clean.
- [ ] The `/v1` compatibility policy and stable error codes are documented.
- [ ] Loopback is the server default; remote bind requires explicit auth.
- [ ] Request body, JSON depth, batch, result, timeout, concurrency, and memory
      limits have enforcement tests.
- [ ] Rust, TypeScript, Python, CLI, and MCP pass the same black-box suite.
- [ ] Framework integrations are optional and import no core internals.

## Supply chain and operations

- [ ] Format, Clippy, tests, doctests, examples, MSRV, and docs are green.
- [ ] RustSec, license/source policy, dependency review, and secret scanning are
      green.
- [ ] Fuzz, fault-injection, load, and soak gates meet recorded thresholds.
- [ ] Release archives contain checksums, SPDX/CycloneDX SBOM, signatures, and
      build provenance.
- [ ] Installation, upgrade, migration, backup, restore, doctor, and verify
      documentation is executable from a clean environment.
